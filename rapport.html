Flux d'authentification (3FA)

1
Facteur 1 – Connaissance  login.php
Email + mot de passe (6–128 caractères) · Blocage 30 min après 3 échecs · Détection de nouvelle IP
2a
Facteur 2a – Questions secrètes  questionnaire.php
3 réponses comparées par hash bcrypt · Insensible à la casse · Toutes obligatoires
2b
Facteur 2b – OTP Email  otp.php
Code 6 chiffres via random_int() · Envoyé par Gmail SMTP · Expire en 90 s · Max 3 tentatives / 3 renvois
3
Facteur 3 – PIN + Biométrie  verify_pin.php
PIN 4 chiffres (bcrypt) · Fallback WebAuthn : Windows Hello, Touch ID, FIDO2 · Algorithmes ES256 / RS256
✓ Authentification complète → dashboard.php

Les flags de session user_id → otp_validated → pin_validated empêchent tout saut d'étape.

Sécurité implémentée

Hachage bcrypt
Mot de passe, PIN et réponses aux questions secrètes – jamais stockés en clair
Requêtes préparées PDO
Toutes les requêtes SQL utilisent des paramètres liés – aucune concaténation
Protection XSS
htmlspecialchars() sur toutes les sorties utilisateur
Tokens haute entropie
256 bits (64 octets hex) pour les tokens de reset et de partage
Fichiers protégés
.htaccess bloque l'accès direct à uploads/ et config/
Noms randomisés
Fichiers renommés avec uniqid() à l'upload – pas de devinette possible
En-têtes HTTP sécurisés
X-Frame-Options, X-XSS-Protection, Referrer-Policy, Permissions-Policy
Anti-replay WebAuthn
signCount incrémental vérifié – prévient la réutilisation de credentials volés
Validation MIME
Type de fichier vérifié par finfo (contenu binaire, pas l'extension)
Blocage de compte
Après 3 tentatives échouées – déblocage automatique après 30 minutes

Points notables

PointDescription
WebAuthn sans dépendance Implémentation complète en PHP natif : décodeur CBOR binaire, parser COSE, vérification de signatures ES256 (ECDSA P-256) et RS256 (RSA) via openssl_verify()
Architecture 3FA robuste Flags de session à 3 niveaux – impossible de sauter une étape d'authentification, même en manipulant l'URL
Zéro accès direct aux fichiers Tous les documents passent par PHP avec vérification de propriété – .htaccess bloque uploads/
Credentials SMTP externalisés Identifiants dans config/credentials.php non versionné – bonne pratique de sécurité
Page pédagogique phishing_demo.php illustre les risques de phishing pour sensibiliser les utilisateurs
Timer de session visible Compte à rebours affiché dans l'interface, alerte colorée dans les 5 dernières minutes
